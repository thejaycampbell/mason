# Mason Example — Next.js App Router SaaS Landing Page

A complete SaaS landing page generated by Mason. Demonstrates the component
architecture, TypeScript patterns, interactivity stubs, and SEO output that
Mason produces for Next.js App Router projects.

**Fictional product:** Dispatch — project management for freelancers.

## What This Example Demonstrates

### Component Architecture
Mason generates a directory of focused components, not a single-file monolith:

```
components/
├── layout/
│   ├── Header.tsx     ← mobile nav with useState toggle
│   └── Footer.tsx
└── sections/
    ├── Hero.tsx        ← LCP image, extractable positioning statement
    ├── SocialProof.tsx ← specific stats (not "thousands of users")
    ├── Features.tsx    ← benefit-led descriptions
    ├── Testimonials.tsx
    ├── Pricing.tsx     ← monthly/annual toggle with useState
    ├── FAQ.tsx         ← accordion with aria-expanded + FAQPage schema
    └── CTA.tsx
```

### TypeScript
Every component has an explicit `interface Props {}` (or named interfaces
for data shapes). No implicit `any`, no inline type unions.

### Interactivity
- **Mobile nav toggle** — `useState`, `aria-expanded`, `aria-controls`
- **Pricing toggle** — monthly/annual switch with `aria-pressed`
- **FAQ accordion** — `useState`, `aria-expanded`, `hidden` attribute,
  `role="region"` for screen readers

### Animation
`useAnimateOnScroll` hook uses `IntersectionObserver` to add the `visible`
class to `.animate-on-scroll` elements as they enter the viewport. Staggered
delays via inline `style` (not dynamic Tailwind classes). Respects
`prefers-reduced-motion`.

### SEO / AEO Output
- **Title, meta description, OG tags** — present on every page
- **Canonical tag** — via `metadata.alternates.canonical`
- **Organization schema** — in root layout (applies to all pages)
- **SoftwareApplication schema** — on homepage alongside Organization
- **FAQPage schema** — injected in FAQ component
- **sitemap.ts** — generates `/sitemap.xml` via Next.js App Router
- **robots.ts** — generates `/robots.txt` with AI crawler allowance
- **next/font** — Inter loaded with zero CLS, self-hosted

## Running It

```bash
cd examples/nextjs-saas
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000).

## Adapting for Your Project

1. Replace Dispatch copy with your product — the brand profile Mason extracts
   from your codebase maps directly to these component slots
2. Update `tailwind.config.ts` colors to match your brand palette
3. Swap `dispatch.so` URLs in `layout.tsx`, `sitemap.ts`, and `robots.ts`
4. Add real images to `public/` and reference them in `Hero.tsx`

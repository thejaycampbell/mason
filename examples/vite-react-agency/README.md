# Mason Example — Vite + React Agency Site

A complete service/agency landing page generated by Mason. Demonstrates the
patterns Mason uses for **Vite + React** projects and **Service/Agency** site types.

**Fictional agency:** Forma — brand and web design studio.

## What This Example Demonstrates

### Service/Agency Section Order
Mason uses this default order for service sites:
```
Hero → Problem → Services → How It Works → Portfolio → Testimonials → FAQ → CTA
```

### Vite + React Patterns
Unlike the Next.js example, this uses Vite-native patterns:
- No `next/image` — uses `loading="lazy"` on images, `loading="eager"` on hero (LCP)
- No `next/font` — uses Google Fonts preconnect in `index.html`
- Code splitting via `React.lazy` + `Suspense` instead of `next/dynamic`
- Organization schema in `index.html` `<script>` tag, not a layout component

### Asset Fallbacks (demonstrating the new v3 pattern)
This project has **zero image assets** in `public/`. Every visual element uses
a designed fallback:

- **`Header.tsx`** — CSS wordmark (`<span>` with brand color) instead of `<img src="/logo.svg">`
- **`Hero.tsx`** — Radial gradient background instead of a hero image
- **`Portfolio.tsx`** — Styled mockup frames instead of screenshots
- **`index.html`** — SVG data URI favicon (first letter of brand name)

### Conversion Architecture
The page follows all rules from mason-builder's Conversion Architecture section:
- Primary CTA in hero — visible above fold at 1280px
- Social proof within 200px of hero CTA (client name bar)
- CTA appears 3× (hero, Services section mid-page, bottom CTA banner)
- Testimonials before CTA (trust signal placement)
- Every testimonial has: quote + name + role + company + result
- Objection-handling line below the bottom CTA button

### FAQ Schema
FAQPage structured data injected in `FAQ.tsx` — answers are co-located with
the schema so they stay in sync. AI engines can extract and cite these directly.

### TypeScript
All components have explicit `interface` definitions. No implicit props.

## Running It

```bash
cd examples/vite-react-agency
npm install
npm run dev
```

Open [http://localhost:5173](http://localhost:5173).

## Key Differences from the Next.js Example

| Feature | Next.js (`examples/nextjs-saas/`) | Vite + React (this) |
|---------|----------------------------------|---------------------|
| Routing | App Router file-based | Single-page (hash links) |
| Images | `next/image` with `priority` | `<img loading="eager">` for LCP |
| Fonts | `next/font/google` | `<link>` preconnect in index.html |
| Code splitting | `next/dynamic` | `React.lazy` + `Suspense` |
| SEO meta | `metadata` export | `<meta>` in index.html |
| Sitemap | `app/sitemap.ts` | `public/sitemap.xml` (static) |
| Schema | Layout component | `index.html` script tag |
| Canonical | `metadata.alternates` | `<link rel="canonical">` in head |
